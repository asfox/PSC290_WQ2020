I am not a Python evangelist. That said, I do use python in my own work.

 

I first learned to code in C, then later in C++ and Java. These languages have too much overhead and are not well suited to data-manipulation programing. For a number of years I used Matlab, but I found it to be great for some things (e.g. matrix math), but terrible for others (e.g. string processing). In the end, Matlab lacked some generality. 

 

I use Python because:

  • Python is free, and open-source
  • Other people use it (most questions can be googled)
  • It does a number of different things well (works with CSVs, matrices, strings, etc.)
  • It tends to be easy to read, and generally follows the rules of a "good" language.
  • It works well with other tools (easily load xlsx, csv, .nii, fasta, etc.)
  • There are a number of useful packages (Numpy, Pandas, Scikit-Learn, Scikit-image, etc.)

In general, these are not things that are specific to Python. In many ways, I think R is also a good language. The main reasons I don't use R are: 1) There are too many ways to do the same thing to be able to read someone else's code, 2) when I made my choice, R was super slow with matrices (though, I've heard this has gotten better), and 3) it can be difficult to transition from R-code to a standalone or web-based program.

In the end, I chose Python for this class, but I encourage you to focus your learning less on the language, syntax can always be googled, and more on the structure, organization, and becoming someone who knows what questions to ask.
