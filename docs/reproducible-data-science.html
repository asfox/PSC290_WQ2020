Part of the reason we organize our code is so that we can re-use it.  

 

Here are some must reads!

Here is a little writeup on how to best use functions for reusability, from UCD's own Titus Brown (he works in the UC Davis genome center). I'd like you to read the sections "Programming for Usability" and the section immediately after it, "Modules and Scripts".

Titus also discusses testing in software. Testing is super useful. In fact, some people believe that this is the most important aspect that is missing from an academic coding education. I'm not going to force you to write tests in your code, but I am going to force you to read through "An introduction to testing concepts" from: http://intermediate-and-advanced-software-carpentry.readthedocs.io/en/latest/testing-python.html

Maybe reads!

If you've still got time, I'd recommend checking out videos on how to do reproducible science in python by Jake VanderPlas, who we saw talk about permutations/shuffling, and who wrote the python data science handbook, has a few videos on how to do reproducible science in python. He's always interesting. :-)
