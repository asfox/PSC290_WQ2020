So, you want to collaborate or share your code with the world? 

A big part of being a responsible scientific citizen is to make the methods that you use available to scrutiny and future use. It used to be that including a "method" section in your manuscript was the best way to do that. I don't believe this is true for most things done with modern computational frameworks. In principle, any code you used to complete your studies should be available to interested researchers. 

To this end, I recommend sharing your code on GitHub. In addition to making your code available to other people for scrutiny, GitHub also enables shared code collaboration. (I even know people who use it for co-writing manuscripts.)

I recommend that you:

  1. Start by creating an account, here. 
  2. Then, you follow through this "Hello world" walkthrough. 

 

Once you get through the basics, you should be ready to share code with each other and with the world. 

 

If you want to take the next step, there are additional GitHub guides, including how to host your own personal web-page (like I do). You can also use the command line to interface with your git, which is generally how I go about it. 

Github is already the norm for people developing open-source software, and is becoming more and more common with researchers who write their own code. In the scientific literature, it has been argued that "to enable and ensure the replicability and traceability of scientific claims, it is essential that the scientific publication, the corresponding datasets, and the data analysis are made publicly available," and that GitHub is the solution.
